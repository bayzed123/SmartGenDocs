How to Use SmartGen Docs

Welcome to the SmartGen Docs user guide! SmartGen Docs is an autonomous, AI-driven documentation platform that transforms any public GitHub repository into a fully deployed, professional documentation site with zero configuration.

This guide will walk you through generating documentation using the automated GitOps workflow (recommended) or running the generator locally.

Method 1: The GitOps Way (Automated & Recommended)

The easiest way to use SmartGen Docs is by utilizing its automated pipeline. You don’t need to install anything locally; you just need to commit a text file containing your target repository link.

Step 1: Copy Your Target Repository Link

Locate the public GitHub repository you want to document. Copy its standard URL:

https://github.com/owner/repository-name

Step 2: Add the Link to input_links/

  1. Navigate to your cloned or forked instance of the SmartGenDocs repository.
  2. Open the input_links/ directory.
  3. Create a new text file with a descriptive name (e.g., my-project-link.txt).
  4. Paste your target GitHub URL directly into the file. Save the file.

Example file content:

https://github.com/bayzed123/SmartGenDocs

Step 3: Commit and Push Changes

Commit the newly created file and push the changes to your main branch:

git add input_links/my-project-link.txt
git commit -m "docs: add new project link for doc generation"
git push origin main

Step 4: Let the GitOps Pipeline Do the Work

Once pushed, the GitHub Actions workflow triggers automatically. The SmartGen Docs Engine will: 1. Validate: Check the URL structure and verify it is not a duplicate. 2. Scan: Analyze the target repository using Gemini 3.5 Flash. 3. Generate: Create 4 distinct documentation pages in the docs/ folder: * Overview: Highlights core features, value proposition, and goals. * Architecture: Focuses on system architecture, code flow, and components. * Deployment Guide: Comprehensive installation and deployment steps. * How to Use: Practical implementation examples and CLI commands. 4. Deploy: Automatically push the rendered HTML pages directly to your live documentation website.

Method 2: Local Generation (For Developers)

If you want to run the generation engine on your local machine to test modifications or run manual scans, follow these steps.

Prerequisites

Make sure you have Python 3.x installed.

Step 1: Clone and Set Up Dependencies

Clone your repository and install the required Python packages:

# Clone the repository
git clone https://github.com/your-username/SmartGenDocs.git
cd SmartGenDocs

# Install dependencies
pip install -r requirements.txt

Step 2: Set Environment Variables

The script relies on GitHub and Google Gemini APIs. Set the environment variables in your terminal (or add them to a local .env file):

# On Linux/macOS
export GITHUB_TOKEN="your_github_personal_access_token"
export GEMINI_API_KEY="your_google_gemini_api_key"

# On Windows (Command Prompt)
set GITHUB_TOKEN="your_github_personal_access_token"
set GEMINI_API_KEY="your_google_gemini_api_key"

# On Windows (PowerShell)
$env:GITHUB_TOKEN="your_github_personal_access_token"
$env:GEMINI_API_KEY="your_google_gemini_api_key"

Step 3: Run the Script

Manually place your target links in the input_links/ folder as text files, then execute the Python engine:

python generate_docs.py

The script will process the text files in input_links/, generate Markdown and HTML assets, and output them to the docs/ directory.

Accessing Your Generated Documentation

Once the deployment pipeline or local execution finishes, your documents will be organized under the following structured layouts:

Page File Destination Description
Overview docs/overview.html Highlights project goals and high-level features.
Architecture docs/architecture.html Provides details on codebase layout and design.
Deployment Guide docs/deployment.html Easy-to-follow setup, hosting, and configuration guides.
How to Use docs/how_to_use.html Code examples, syntax guides, and runtime configurations.

If you have GitHub Pages enabled on your repository, these documents will immediately display live on your site's domain (e.g., https://<owner>.github.io/SmartGenDocs/).
