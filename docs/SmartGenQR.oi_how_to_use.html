2. For Developers: Local Setup & Installation

If you want to run SmartGen locally, customize the web utilities, or test your modifications locally, follow these steps.

Prerequisites

  • Node.js installed on your machine.
  • pnpm package manager (preferred).

Step-by-Step Installation

  1. Clone the Repository: bash git clone https://github.com/bayzed123/SmartGenQR.oi.git cd SmartGenQR.oi

  2. Install Dependencies: bash pnpm install

  3. Run Locally: Since the platform is 100% client-side, you can open any index.html file directly in your browser. However, to experience the full routing environment, run a local development server: bash npx serve Access the local site at the URL output by the terminal (typically http://localhost:3000 or http://localhost:5000).

3. For Content Managers: Writing & Building Blog Posts

SmartGen uses an automated, static blog builder that converts Markdown files into static, highly optimized web pages.

Step 1: Write your Blog Post

Create a new Markdown (.md) file inside the /blog-posts/ directory. Be sure to use YAML front matter at the top of the file to declare metadata:

title: "10 Essential SEO Strategies for 2026"
date: "2026-03-30"
description: "Discover the top tips and tricks to optimize your client-side web utility performance."

Step 2: Build the Blog

Generate static pages and compile the global blog index JSON by running the build script:

pnpm build

(This triggers node scripts/build-blog.js using marked, front-matter, and slugify to render the changes).

4. For Contributors: Creating & Registering a New Tool

To scale the catalog of 45+ utilities while maintaining performance, please adhere strictly to our development standards:

Standard Checklist

  1. Directory Structure: Every tool must reside in its own isolated sub-folder containing its entry point file: text /my-new-utility-tool/index.html
  2. Logic Separation: Keep the UI clean. Write your operational code inside pure vanilla JavaScript. No external server frameworks (Node, Express, etc.) are allowed for tool execution.
  3. SEO Skyscraper Rule: Each tool's index.html must contain 1200+ words of helpful documentation, instructional steps, FAQ Schema markup, and meta tags for search engine optimization.
  4. Registration in Search Index (Crucial!): For your tool to be discoverable via the platform search bar and recommended widgets, open assets/js/search-data.js and add your tool's details to the global TOOLS_INDEX array: javascript // Example Entry { name: "My New Utility Tool", path: "/my-new-utility-tool/", category: "Developer", // Technical, Marketing, or SEO tags: ["conversion", "parser", "format"] }
  5. Test Locally: Run npx serve to verify that your tool functions perfectly and displays correctly in the dynamic recommendation system managed by assets/js/related-tools.js.
