SmartGen Documentation Hub: Deployment Guide

This guide provides comprehensive instructions for deploying, hosting, and maintaining the SmartGen Documentation Hub—a GitOps-powered documentation site built with MkDocs Material.

1. System Architecture Overview

The SmartGen Documentation Hub uses a Zero-Touch GitOps deployment pipeline. The architecture is decoupled into three distinct phases:

[ Local Development ] ────( Push to main )────> [ GitHub Repository ]
                                                       │
                                            (GitHub Actions Triggered)
                                                       ▼
[ GitHub Pages / Web Server ] <───( Deploy )─── [ Build & Artifacts ]
  • Static Site Generator (SSG): MkDocs Material.
  • Asset Pipeline: Custom Python scripts executing dynamic social card generation (Open Graph images) using PIL (Pillow) and CairoSVG.
  • CI/CD Pipeline: GitHub Actions automation compiling Markdown source files into optimized HTML/CSS/JS and deploying them automatically.

2. Environment Prerequisites

Ensure the target environments (local development and runner/production) meet the following requirements:

System Requirements

  • Operating System: Linux, macOS, or Windows 10/11.
  • Runtime Environment: Python 3.8 or higher.
  • System Dependencies (for Social Cards):
    • libcairo2 (Required by CairoSVG for rendering vector graphics to PNG).

Installing System Dependencies

  • macOS (via Homebrew): bash brew install cairo pango gdk-pixbuf libffi
  • Ubuntu/Debian: bash sudo apt-get update sudo apt-get install -y libcairo2 libpango-1.0-0 libgdk-pixbuf2.0-0 libffi-dev shared-mime-info
  • Windows: Download the appropriate GTK/Cairo binaries or use MSYS2.

3. Local Development & Verification

Before deploying changes to production, verify your modifications locally.

Step 1: Clone the Repository

git clone https://github.com/bayzed123/docslab.git
cd docslab

Step 2: Establish a Virtual Environment (Recommended)

# Create virtual environment
python3 -m venv venv

# Activate virtual environment
# On macOS/Linux:
source venv/bin/activate
# On Windows (Command Prompt):
venv\Scripts\activate.bat

Step 3: Install Dependencies

Install the package requirements alongside the optional graphics rendering engines:

pip install -r requirements.txt
pip install pillow cairosvg

Step 4: Run the Local Live-Reload Server

mkdocs serve
  • Local Access URL: http://127.0.0.1:8000
  • Note: Any changes made to the Markdown files (.md) inside the docs/ directory will compile and reload in the browser instantly.

4. Continuous Integration & Deployment (CI/CD)

The project utilizes GitHub Actions to automate production deployment. When a pull request is merged or a direct commit is pushed to the main branch, the deployment workflow triggers automatically.

Configuring the GitHub Actions Workflow

Create or verify the deployment workflow file under .github/workflows/deploy.yml:

name: Deploy SmartGen Documentation Hub

on:
  push:
    branches:
      - main
  workflow_dispatch: # Allows manual trigger of deployments

permissions:
  contents: write
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0 # Important for Git revision dates inside MkDocs

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-name: '3.10'
          cache: 'pip'

      - name: Install System Dependencies for CairoSVG
        run: |
          sudo apt-get update
          sudo apt-get install -y libcairo2-dev libpango1.0-dev libgdk-pixbuf2.0-dev libffi-dev shared-mime-info

      - name: Install Python Dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install pillow cairosvg

      - name: Execute Custom Data Aggregation Scripts
        run: |
          if [ -d "./scripts" ]; then
            python scripts/aggregate.py || echo "No scripts or execution failed, continuing build..."
          fi

      - name: Build MkDocs Site
        run: |
          mkdocs build --clean

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./site

5. Deployment Checklist & Verification

Initial GitHub Repository Setup

  1. Go to your GitHub repository: https://github.com/bayzed123/docslab.
  2. Navigate to Settings > Pages.
  3. Under Build and deployment, set the Source to Deploy from a branch (typically configured automatically by the actions runner using the gh-pages branch).
  4. Verify that the Custom Domain (if any) is configured and SSL is forced.

Post-Deployment Validation

Once a pipeline run completes successfully: * [ ] Verify the SSL certificate of the deployed website is valid. * [ ] Test dynamic Open Graph social share images by running an inspection tool like the Facebook Sharing Debugger or Rich Preview. * [ ] Confirm that files generated dynamically via scripts inside the /scripts directory have aggregated correctly into the production site layout.

6. Troubleshooting Deployment Failures

Symptom Probable Cause Resolution
Workflow failure at CairoSVG/Pillow step Missing OS-level shared system library binaries on the build agent. Ensure the Install System Dependencies for CairoSVG step in .github/workflows/deploy.yml runs successfully before executing Python commands.
404 Page Not Found on live URL Custom domain configuration issue or GitHub Pages is pointed to the wrong branch. Check repository Settings > Pages and ensure the deployment source is mapped to the gh-pages branch.
Build passes but styling/theme is broken Incorrect configuration of the site_url property in mkdocs.yml. Open mkdocs.yml and verify that the site_url key reflects the absolute base URL of your deployed domain (e.g., https://bayzed123.github.io/docslab/).
