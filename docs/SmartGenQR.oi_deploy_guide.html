🚀 Deployment & Hosting Guide: SmartGen

This guide provides comprehensive, step-by-step instructions for deploying, hosting, and maintaining the SmartGen All-in-One Digital & Web Utility Platform.

Since SmartGen is built on a 100% Client-Side Architecture, hosting is highly cost-effective and extremely fast. It does not require database setups or server-side runtimes (like Node.js, Python, or PHP) on the production server.

1. System Architecture Overview

SmartGen runs entirely on the browser. However, it contains a static compilation step for its Markdown-based blog.

[Markdown Posts] ──(pnpm build)──> [Static Blog Pages] 
                                            │
[Tool Directories (HTML/JS)] ────────────────┼──> [Distribution Target / Web Host]
                                            │
[Global Assets (CSS/Shared JS)] ────────────┘
  • Frontend Stack: HTML5, Tailwind CSS/Vanilla CSS, Vanilla JavaScript.
  • Blog Engine: Static node-based build system using Markdown, marked, front-matter, and slugify.
  • Execution Environment: Modern web browsers (No backend/database required).

2. Prerequisites

Ensure you have the following installed on your local machine or build agent: * Node.js: v18.x or higher (Required only for the blog compilation script). * Package Manager: pnpm (Highly recommended, used in the original project setup). * Git: For version control and deployment hooks.

3. Local Development & Pre-Deployment Build

Before deploying to any platform, run a local build to generate the updated static blog pages, search index, and metadata files.

Step 1: Clone the Repository

git clone https://github.com/bayzed123/SmartGenQR.oi.git
cd SmartGenQR.oi

Step 2: Install Dependencies

pnpm install

Step 3: Run the Compilation Script

If you have written new blog posts under /blog-posts/, generate the static pages and the index file (blog.json) by running:

pnpm build

(This triggers node scripts/build-blog.js as defined in package.json).

Step 4: Verify the Registry Update

Ensure any newly deployed tools are added to the search index registry so users can access them: * Open assets/js/search-data.js and ensure the TOOLS_INDEX array contains your new directory.

Step 5: Test Locally

You can run a quick local HTTP server to verify the build output:

npx serve .

Access the application at http://localhost:3000.

4. Deployment Target Options

Choose the hosting option that best matches your infrastructure.

Option A: GitHub Pages (Recommended / Automated)

SmartGen is fully integrated with GitHub Actions. Any push to the main branch will automatically trigger a build and publish to GitHub Pages.

Setup Instructions:

  1. Push your cloned code to your GitHub repository.
  2. Go to your repository on GitHub.
  3. Navigate to Settings > Pages.
  4. Under Build and deployment:
  5. Source: Select Deploy from a branch (Or select GitHub Actions if you want to use custom action-based compilation).
  6. Branch: Select main and set folder to / (root).
  7. Click Save.

Your deployment workflow status can be viewed directly under the Actions tab of your repository.

Option B: Cloudflare Pages / Vercel / Netlify

These modern edge environments offer global CDN distribution, automatic SSL, and instant rollbacks.

Configuration Reference:

  • Build Command: pnpm build
  • Publish Directory: . (or root directory)
  • Node.js Version: 18 or 20
Platform Build Setting: Build Command Build Setting: Output Directory
Cloudflare Pages pnpm build .
Vercel pnpm build . (Override setting to use Root /)
Netlify pnpm build .

Option C: Standard Nginx / Apache Server

For self-hosting on virtual private servers (VPS) or shared hosting.

Configuration Checklist:

  1. Run pnpm build locally.
  2. Upload the entire project root directory to your web server’s document root (e.g., /var/www/html/ or public_html/).
  3. Nginx Configuration: Ensure Nginx handles routing correctly. Use the following block configuration:
server {
    listen 80;
    server_name smartgentools.com www.smartgentools.com;
    root /var/www/html;
    index index.html;

    # Dynamic search friendly routing (Clean URLs)
    location / {
        try_files $uri $uri/ $uri.html =404;
    }

    # Cache static assets aggressively
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, no-transform";
    }

    # Ensure system files / .git are secure
    location ~ /\. {
        deny all;
    }
}

5. Configuring Custom Domains & SSL

To configure your own branding (e.g., smartgentools.com):

1. DNS Records Setup

Add the following DNS records at your domain registrar:

Record Type Host Value TTL
A @ Points to your hosting provider's IP Automatic
CNAME www Points to your hosting provider's default alias Automatic

2. Force SSL (HTTPS)

  • GitHub Pages: Check the "Enforce HTTPS" box under the Pages Settings panel.
  • Cloudflare Pages/Vercel: SSL is provisioned and enforced automatically by default.
  • Nginx Server: Utilize Let's Encrypt Certbot to obtain a free SSL certificate: bash sudo apt install certbot python3-certbot-nginx sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com

6. Post-Deployment Validation & Troubleshooting

Once deployed, execute these sanity checks to ensure the application is functioning correctly:

🔍 Validation Checklist

  • [ ] Functional Search Engine: Type a tool name (e.g., "QR") into the search bar. If no suggestions appear, verify that assets/js/search-data.js has been correctly updated and loaded.
  • [ ] Blog Posts Loading: Navigate to /blog and verify that latest blog posts are visible. If posts return a 404, verify that pnpm build successfully created the HTML files in the output directory.
  • [ ] HTTPS Redirection: Verify that attempts to access http:// redirect automatically to https://.
  • [ ] Sitemap and SEO Assets: Ensure sitemap.xml exists in the root directory and contains links pointing to your custom domain instead of the default GitHub Pages subpath.

🛠️ Common Issues & Resolutions

  • Issue: "Failed to load module / JS files return 404 in sub-pages"
    • Fix: SmartGen uses relative directory trees. Avoid hardcoded absolute root-slashes (/assets/js/...) in individual tool directories. Ensure the paths are configured as relative (../assets/js/...) to maintain compatibility with subfolders.
  • Issue: "Blog updates aren't showing up on production"
    • Fix: Make sure scripts/build-blog.js ran successfully before git push. Ensure data/blog.json has been updated with your new post's metadata.
