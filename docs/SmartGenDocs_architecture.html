SmartGen Docs — System Architecture Guide

SmartGen Docs is an autonomous, zero-configuration, GitOps-driven documentation platform. It leverages Google Gemini 3.5 Flash to transform public GitHub repositories into rich, production-ready, multi-page documentation sites.

This guide outlines the system architecture, component design, data flow, and underlying technology stack of the SmartGen Docs platform.

2. Component Breakdown

The system is organized into decoupled layers, utilizing a file-based trigger mechanism to coordinate pipeline executions.

A. The GitOps Trigger (input_links/)

  • Role: Serves as the system interface and scheduling mechanism.
  • Mechanism: A text file (e.g., project-link.txt) is placed inside input_links/ containing a public GitHub URL. A push event to this directory signals the GitHub Actions continuous integration (CI) workflow.

B. CI/CD Orchestration (.github/workflows/)

  • Role: Automates environment creation, dependency resolution, execution scripts, and Git deployment.
  • Workflow Actions:
    1. Listens for file changes in the input_links/ directory.
    2. Sets up Python 3.x environment.
    3. Installs dependencies from requirements.txt.
    4. Executes generate_docs.py using repository-scoped environment secrets (e.g., GEMINI_API_KEY and GITHUB_TOKEN).
    5. Commits and pushes generated outputs back to the target branch.

C. The Core Engine (generate_docs.py)

This is the central execution file containing the core platform logic. It is structured into three main phases: 1. Validator: Parses input files, filters out duplicates, and validates that target repository URLs match the standard pattern (https://github.com/owner/repository). 2. Repository Parser: Uses PyGithub to scan the codebase structure, directory hierarchy, config files (e.g., package.json, cargo.toml, go.mod), and the default README.md to compile a contextual digest of the target project. 3. Markdown-to-HTML Compiler: Uses the markdown Python library to process the generated markdown files into standard, responsive HTML structures, updating the static site folder (docs/) and generating landing pages (index.html).

D. AI Ingestion Layer (Google Gemini 3.5 Flash)

  • Role: Performs source code analysis and synthesizes specialized developer documentation.
  • Integration: Powered by the official google-generativeai client.
  • Why Gemini 3.5 Flash?
    • Extremely Low Latency: Fast response times keep documentation builds under the GitHub Actions execution limit.
    • Generous Context Window: Allows the engine to ingest larger file structures and configuration files directly.
    • Code Understanding: Excellent native capabilities for parsing directory trees, config files, and standard code architecture.

Detailed Execution Steps

Step 1: Input & Ingestion

The process begins when a developer writes a plain-text file into input_links/:

https://github.com/bayzed123/SmartGenDocs

A GitHub Action trigger detects this update and passes execution to generate_docs.py.

Step 2: Context Extraction

The engine utilizes PyGithub to construct a project topology footprint: * It reads README.md to extract the primary mission statement. * It crawls the root structure to find configuration files (package.json, requirements.txt, etc.). * It constructs a representation of the file tree to determine the framework and language patterns.

Step 3: LLM Inference

The collected project metadata is formatted into specialized system prompts sent to Gemini 3.5 Flash. The engine requests 4 distinct functional deliverables: 1. Overview: High-level value propositions, goals, and architectural summaries. 2. Architecture: Technical structure, framework analysis, and directory breakdowns. 3. Deployment Guide: Targeted steps to host, configure, and package the application. 4. How to Use: Practical instructions, API commands, and CLI examples.

Step 4: Page Building & Compilation

The incoming markdown payloads are parsed through the Python markdown library and wrapped in an HTML layout wrapper (featuring clean typography, responsive mobile grids, and side-navigation matching the main index.html UI scheme).

Step 5: Version Controlled Deployment

The workflow commits the resulting assets (docs/*.html, updated index, and static assets) back to the GitHub Repository. This triggers GitHub Pages to deploy the updated documentation instantly.

5. Security, Validation & Resiliency

To prevent infinite loops, API overruns, and broken links, the SmartGen Docs architecture implements several guardrails:

  • Idempotency & Deduplication: The engine compares incoming URLs against a log of already processed repositories. If a link has already been built and no upstream changes have occurred, the build pipeline exits early.
  • Strict Pattern Matching: Validates target links via regular expressions, filtering out malformed, private, or invalid URL schemes before making external network calls.
  • Secret Masking: The GEMINI_API_KEY and GITHUB_TOKEN are injected as repository secrets, ensuring they are never exposed in build logs or committed metadata.
  • Graceful API Rate-Limiting: Uses chunked read operations via PyGithub and processes LLM operations with structured error handling to manage Gemini API rate limits gracefully.
