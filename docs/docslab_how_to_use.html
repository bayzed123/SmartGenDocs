SmartGen Documentation Hub: User & Contributor Guide

Welcome to the SmartGen Documentation Hub! This guide will walk you through setting up the documentation environment locally, writing or updating content, and publishing your changes.

The SmartGen documentation is built using MkDocs Material and powered by a GitOps pipeline. Any updates you push to the repository are automatically built and deployed live.

📋 Prerequisites

Before you begin, ensure you have the following installed on your local machine: * Python 3.8 or higher * Git

🛠️ Step 1: Local Development Setup

To preview changes on your local machine before pushing them to the live site, follow these setup steps:

1. Clone the Repository

Open your terminal and clone the repository, then navigate into the project directory:

git clone https://github.com/bayzed123/docslab.git
cd docslab

2. Install Dependencies

Install MkDocs Material and the required libraries for processing images and dynamic social cards:

Using the manual pip command:

pip install mkdocs-material pillow cairosvg

(Alternative) Or install via the provided requirements.txt file:

pip install -r requirements.txt

3. Launch the Local Server

Run the built-in live-reload server to preview your site locally:

mkdocs serve

4. Preview the Site

Once the server starts, open your web browser and navigate to: 👉 http://127.0.0.1:8000

Note: The local server features live-reload. Any saving of markdown files will instantly update in your browser.

✍️ Step 2: How to Edit and Update Content

All documentation content is stored in the docs/ directory as Markdown (.md) files.

  1. Navigate to the docs/ folder in your preferred text editor.
  2. To Edit Existing Pages: Locate and open the relevant .md file, make your changes, and save.
  3. To Create New Pages: Create a new Markdown file (e.g., features.md) within the docs/ directory. Remember to register new files in the navigation section of the mkdocs.yml configuration file if structure modifications are required.

🚀 Step 3: Publishing Your Changes

The documentation site uses a Zero-Touch GitOps deployment flow. Once you are happy with your local changes, publish them by following these steps:

  1. Stage and commit your changes: bash git add . git commit -m "docs: update SmartGen user guide"

  2. Push to the main branch: bash git push origin main

  3. Automatic Deployment: Upon pushing, a GitHub Actions workflow is automatically triggered. It will:

  4. Build the production static site.
  5. Generate dynamic Open Graph social cards.
  6. Deploy the updated build to your hosting server within seconds.

📁 Repository Structure Overview

To help you navigate the codebase, here is the directory layout:

  • docs/: Holds the Markdown files containing the actual documentation text.
  • mkdocs.yml: The main configuration file (controls theme, navigation, and plugins).
  • overrides/: Custom HTML/CSS templates to override default MkDocs Material layouts.
  • layouts/: Source assets and background images for dynamic social cards.
  • scripts/: Python utility scripts that aggregate external API data or guidelines.
