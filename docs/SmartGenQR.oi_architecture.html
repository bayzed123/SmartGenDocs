⚡ SmartGen Architecture Guide

This guide details the architectural design, directory structure, data flow, and deployment pipeline for SmartGen, a high-performance, privacy-first web utility platform.

1. Architectural Principles & Philosophy

SmartGen is built on three core pillars: Zero-Backend Execution, Search Engine Dominance (SEO), and Instantaneous Load Times.

Key Architectural Traits

  • 100% Client-Side Computation: Every single tool (e.g., QR Generator, JSON Formatter, Word Counter) processes data locally inside the user's browser. No user payload or personal data is sent to external servers. This satisfies severe privacy mandates (GDPR/CCPA compliance by design).
  • Decentralized Tool Isolation: Rather than serving a monolithic single-page application (SPA), each utility is decoupled into its own routing directory (e.g., /qr-generator/index.html). This prevents a bug in one tool from causing platform-wide runtime failures.
  • SEO-First Indexation: Every tool's entry page acts as an independent marketing landing page ("SEO Skyscraper"), structured with structural schema and deep copy to rank for organic search without heavy dynamic hydration.

4. Subsystem Deep-Dive & Component Design

A. The Decentralized Tool Pattern

Every utility follows a strict design blueprint. There are no build steps for individual tools; they run on standard vanilla browser constructs.

B. Client-Side Search & Recommendation Routing

To prevent external API calls for simple operations like searches, SmartGen uses a localized indexing network: 1. The Core Index (search-data.js): Contains the master TOOLS_INDEX object containing metadata, tags, and pathways for all tools. 2. The Query Engine (app.js): Intercepts client search-bar keystrokes and runs a fuzzy match algorithm locally against TOOLS_INDEX. 3. The Recommendations Matrix (related-tools.js): Calculates categorization distance during the page-load lifecycle and injects related tools directly into the DOM based on the current context.

5. Build, CI/CD, and Deployment Lifecycle

SmartGen shifts compilation tasks away from the client or target hosting servers. The environment relies entirely on standard DevOps runners to execute validation checks and generate build artifacts.

[ Developer Push to main ]
          │
          ├──> GitHub Action: Auto-Changelog Workflow
          │    ├── Generates automated changes from commits
          │    └── Updates and commits `data/changelog.json`
          │
          └──> GitHub Action: Pages Deploy Workflow
               ├── Checks out code & configures Node.js (pnpm environment)
               ├── Runs `build-blog.js` compilation step
               └── Deploys static distribution workspace directly to GitHub Pages CDN

Build & Dependency Pipeline

Production runs do not require heavy JS transpilation frameworks (like Webpack or Vite), keeping runtime performance close to raw HTML rendering speeds. The development environment uses the following configuration:

Developer Standards Check List

  1. Strict Sandboxing: Do not import scripts from other tools. Keep business logic contained within your tool's directory.
  2. SEO Skyscraper Blueprint: The landing page index.html must contain 1200+ words of helpful content and valid JSON-LD schemas representing common questions and answers (FAQ schemas).
  3. No External Libraries via CDNs: To protect user privacy, do not load third-party trackers or libraries from external endpoints. Use vendor scripts stored directly within the codebase when possible.
