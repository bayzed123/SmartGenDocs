How to Use: Auto Sitemap Generator

The Auto Sitemap Generator is an automated crawler and backend service that systematically maps web properties, respects robots.txt exclusion rules, and generates valid, SEO-compliant XML sitemaps using a Breadth-First Search (BFS) engine.

You can use this tool in two ways: via the Web-Based Live Demo (no installation required) or locally on your machine (for developers/custom crawls).

Method 1: Using the Web-Based Dashboard (Quick & Easy)

If you don't want to run any code locally, you can use the pre-built web interface.

  1. Access the Tool: Go to the Sitemap Generator Live Demo.
  2. Enter the Target URL: Input the homepage URL of the website you want to map (e.g., https://example.com).
  3. Start the Process: Click the initiate/start button to let the crawler run.
  4. Monitor Progress: Watch the real-time visual progress updates on the dashboard.
  5. Download the Sitemap: Once complete, download the generated sitemap.xml directly to your computer.

Method 2: Running Locally (For Developers)

If you want to customize crawl behavior, run crawls on local development environments, or integrate the process into your own pipelines, run it locally.

1. Prerequisites

Ensure you have the following installed: * Node.js (v14 or higher recommended) * Git

2. Setup & Installation

Open your terminal and run the following commands:

# Clone the repository
git clone https://github.com/bayzed123/SitemapGenerator.git
cd SitemapGenerator

# Install dependencies (This will also install Puppeteer)
npm install

3. Configure the Crawler

Before running the crawler, you must set your target (seed) URL: 1. Open the project folder in your code editor. 2. Open backend/crawler.js. 3. Locate the target seed URL configuration variable (usually at the top or inside the initialization function of the script) and replace it with your desired domain: javascript // Example configuration inside backend/crawler.js const SEED_URL = "https://yourwebsite.com";

4. Run the Crawler

Execute the application using either of the following commands:

# Using the npm start script
npm start

# OR running the file directly
node backend/crawler.js

5. Accessing the Output

Once the process is complete, look inside the generated output/ directory:

  • sitemap.xml: Your production-ready SEO sitemap. Upload this to your website's root directory and submit it to Google Search Console.
  • links.json: A raw list of all discovered and processed links.
  • progress.json: A log file tracking the status of the crawl state.

Key Features to Keep in Mind

  • Robots.txt Compliance: The crawler automatically fetches and parses the target website's robots.txt file. If a path is blocked in robots.txt, the crawler will skip it to comply with SEO best practices.
  • Avoids Infinite Loops: Using in-memory visited-state tracking, the BFS engine guarantees that no page is crawled or logged twice.
  • JavaScript Support: Because the project relies on Puppeteer, the crawler can successfully navigate modern, JavaScript-rendered single-page applications (SPAs).
